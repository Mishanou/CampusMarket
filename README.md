# CampusMarket — Студенческий Мини-Маркетплейс

**CampusMarket** — это веб-приложение, представляющее собой мини-маркетплейс для студентов. Здесь пользователи могут публиковать свои товары, просматривать общий каталог объявлений и сообщать продавцу о желании приобрести товар. Проект реализован с использованием гибридной микросервисной архитектуры, демонстрирующей интеграцию традиционного SSR (Server-Side Rendering) и реактивного асинхронного API с Realtime-уведомлениями.

---

## Архитектура и стек технологий

Проект разделен на зоны ответственности между двумя основными бэкенд-сервисами, общающимися асинхронно через брокер сообщений:

* **Клиентская часть & Маршрутизация (Nginx):** Выступает в роли единой точки входа (Reverse Proxy), которая бесшовно распределяет запросы: пути `/api/...` и `/docs` проксируются на FastAPI, а остальные запросы уходят на Laravel.
* **Основной сервис (Laravel 13 / PHP-FPM):** Отвечает за транзакционную логику, безопасность и SSR-страницы.
    * Аутентификация и сессии (Laravel Breeze + Alpine.js).
    * Панель продавца (CRUD-управление собственными товарами, защищенное `@csrf` и middleware `auth`).
    * Инициация событий: при "покупке" заказа выполняет `Redis::publish`.
* **Микросервис данных и реалтайма (FastAPI / Uvicorn):** Высокопроизводительный асинхронный сервис на Python.
    * Предоставляет REST API для быстрого чтения каталога товаров и карточек без нагрузки на основной сервис.
    * Поддерживает постоянные WebSocket-соединения с клиентами продавцов по адресу `/ws/{user_id}`.
    * В фоновом асинхронном потоке подписан на Redis Pub/Sub, перехватывает сообщения о заказах и мгновенно пушит их в WebSocket-каналы соответствующих продавцов.
* **Фронтенд-компоненты (React / CSR-острова):**
    * Интегрирован внутрь Blade-шаблонов Laravel.
    * **Каталог товаров:** Рендерится на стороне клиента (CSR), запрашивая JSON-данные напрямую у FastAPI (`GET /api/products`).
    * **Realtime-уведомления:** JS-клиент на страницах продавца держит связь с FastAPI по WebSocket и выводит интерактивные всплывающие окна (Toast) при поступлении новых заказов без перезагрузки страницы (F5).
* **База данных и кэш:**
    * **MySQL:** Единая БД (таблицы `users`, `categories`, `products`, `orders`). Laravel имеет полные права (чтение/запись), FastAPI обращается к БД асинхронно в режиме Read-Only.
    * **Redis:** Используется как брокер сообщений (Pub/Sub) для связывания Laravel и FastAPI без использования синхронных HTTP-callback'ов.

## Развертывание проекта в Docker Compose (Windows)

```bash
git clone https://github.com/Mishanou/CampusMarket campus-market
cd campus-market
copy laravel\.env.example laravel\.env
copy fastapi\.env.example fastapi\.env
docker compose up --build -d
docker exec cm_laravel php artisan key:generate
docker exec cm_laravel php artisan migrate --force --seed
```

## Основные сценарии

После успешного выполнения шагов развертывания доступны следующие эндпоинты:

* **Главная страница (Витрина маркетплейса):** http://localhost — доступна всем пользователям (включая гостей). Сетка товаров подгружается через React-компонент асинхронно из FastAPI.
* **Авторизация и ЛК:** Ссылки Login / Register в шапке сайта. Позволяют войти в систему для добавления товаров в корзину и перехода в Панель продавца (/products), где доступен CRUD товаров. Имеется возможность авторизации через GitHub.
* **Интерактивная документация API:** http://localhost/docs — автоматически сгенерированный Swagger UI от сервиса FastAPI, позволяющий протестировать эндпоинты чтения каталога.

* **Проверка цепочки Realtime-уведомлений**
Чтобы убедиться в работоспособности асинхронной связи через Redis Pub/Sub и WebSocket, проведите следующий тест:
* Откройте приложение в двух разных браузерах (или обычном режиме и инкогнито).
* В первом браузере войдите под учетной записью Пользователя А (Продавец) и создайте новый товар.
* Во втором браузере войдите под учетной записью Пользователя Б (Покупатель), перейдите на главную страницу и нажмите "Купить" товар Пользователя А.
* Результат: В первом браузере на экране Продавца мгновенно появится всплывающее интерактивное окно с информацией о новом заказе. Страница при этом не перезагружается.
